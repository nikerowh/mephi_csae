# **Основы С**
### **Самая простая программа**
- Самое простое, что можно написать на С, это вывод сообщения в консоль. 
- Сначала включаем информацию о стандартной библиотеке ввода/вывода (`stdio.h` - standard input/output header — библиотека, содержащая некоторый набор стандартных функций, например ввода/вывода). 
- Затем объявляем функцию `int main()`, - точка входа в программу. Самая по себе функция `main()` должна быть в программе в 99% случаев, так как в ней происходит работа самой программы. Причем `main()` имеет тип данных `int`, то есть результатом программы по сути является целое число. Внутри { операторных скобок } находится тело функции, где описано все, что выполняет функция. А раз функция `main()` - это основная функция программы в целом, значит в `main()` по сути описана работа всей программы.
- После вызываем функцию `printf("Hello world")`, выводим сообщение. Тут важно понять, что `printf()` это вывод информации в консоль, и сама функция берется из подключаемой библиотеки `<stdio.h>`, про которую было написано ранее. В качестве аргумента функции (здесь вспоминаем 9 класс в школе, где изучались функции, например $f(x)$, где $f$ - название функции, а $x$ - аргумент, то есть мы что-то даем функции, и получаем от нее какой-то результат, то есть *функция возвращает результат*).
- Но поскольку функция `main()` возвращает целое число, то пишем `return 0;`. `return` - это так называемое ключевое слово, которое обозначает, что функция возвращает результат, которым в данном случае является просто число 0. Все просто.
```
#include <stdio.h>

int main()
{
	printf("Hello world");
	return 0;
}
```

### **Комментарии**
- **Комментарии** нужны банально для того, чтобы потом легче читать код, особенно когда давно не писал программу. В комментариях можно написать что угодно: как работает функция, для чего она нужна, какая и для чего переменная нужна. 
- Комментарии не читаются компилятором (программой, которая переводит код на языке программирования в машинный код, который "скармливается" процессору, после чего программа выполняется). То есть, что бы мы не написали в комментарии, они не будут выполняться и читаться компилятором. Нужны они просто для описания работы кода или каких-то конкретных блоков кода как для себя, так и для других программистов, которые могут прочитать код.
- Есть два вида комментариев: **однострочный** и **многострочный**. Ниже описан синтаксис их написания в коде:
```
// однострочный комментарий
/*
	много
	строчный
	комментарий
*/
```

### **Экранирование строки**
- **Экранирование строки** - это процесс замены управляющих (служебных) символов на соответствующие им последовательности обычных символов. Например, табуляция или же отступ - это 4 пробела. Чтобы не писать в строке 4 пробела придумали управляющий символ, который делает это автоматически при написании его в строке. 
- Ниже приведены варианты экранирования строки:
- `\n` - перенос строки.
- `\t` - табуляция (отступ). Изначально равна 4 пробелам.

### **Основные типы данных**
- **Типы данных** - это классификация данных, которые используются в программах. Они определяют: какие значения может принимать переменная (символ, число, строка т. д.), какие операции можно выполнять с этими значениями и сколько памяти в компьютере занимает переменная. Типы данных помогают компьютеру правильно работать с информацией и определять ее, предотвращая ошибки. 
- Ниже приведены основные типы данных в языке С:
- `int`  - целое число
- `float` - число с плавающей точкой (дробное число)
- `char`  - символ, или же единичный байт
- `short int`  - короткое целое число
- `unsigned short int` - короткое неотрицательное число
- `long`  - длинное целое число
- `double`  - число с плавающей точкой с двойной точностью
- `register int` - тип данных для объявление регистровой переменной
- `void` - тип без значения
- `bool` - логический тип данных, принимает значение "истина" или "ложь" (`true` или `false`)

### **Объявление переменных**
- Переменные используются для хранения некой информации, представленной в виде определенного типа данных. 
- Переменные можно сначала объявить (создать), потом присвоить им значение (определить). Можно сделать все и сразу. 
- В первом случае объявляем переменную по синтаксическому шаблону `<тип данных> <имя переменной>;`. Примеры:
```
int age;
float x;
char symbol;
```
- В примере выше мы определили целочисленную переменную `age`, вещественную переменную `x`, символьную переменную `symbol`.
- Во втором случае то же самое, только через знак "равно" присваиваем значение. Продолжу пример с прошлыми переменными:
```
int age = 18;
float x = 42.74;
char symbol = '@';
```
- Однако важно сказать, что в С есть ряд ключевых (зарезервированных) слов, которые **нельзя** использовать в качестве названия переменной или функции, так как они уже являются частью синтаксиса языка С.  Вот они, все слева направо:
```
auto, break, case, char, const, continue, default, do, double, else, enum, extern, float, for, goto, if, int, long, register, return, short, signed, sizeof, static, struct, switch, typedef, union, unsigned, void, volatile, while
```

### **Математика**
- Базовые математические операции это сложение, вычитание, умножение, деление (с целыми числами отбрасывает дробную часть, то есть после дробной точки, с вещественными числами деление проходит как обычно) и остаток от деления:
```
int x = 34, y = 7;

printf(x + y); // 41
printf(x - y); // 27
printf(x * y); // 238
printf(x / y); // 4
printf(x % y); // 6
```
- Хочется большего? Подключай библиотеку `#include <cmath>`, от возведения в степень до тригонометрии, в большинстве случаев этого хватает. Мало - пиши функции сам, будь самостоятельным.

### **Библиотеки**
- **Библиотека** - это готовый набор функций, классов, методов, подпрограмм для какого-то языка программирования. Обычно такие наборы объединены назначением или сферой использования, например для веб-программирования, разработки нейросетей или для математики.
- Подключение библиотеки в языке С определяется следующим образом:
```
#include <название_библиотеки>
```
- В шаблоне выше сначала идет ключевое слово `#include`, обозначающее "включение" библиотеки в программу для ее дальнейшего использования. Затем прописывается название библиотеки, причем острые скобки сохраняются, название библиотеки пишется прям в них.

### **Ввод и вывод**
- Для ввода/вывода в консоли используется библиотека `<stdio.h>`:
```
#include <stdio.h>
```
- Для вывода в консоль пишется `printf("Hello world!";\n)`
```
#include <stdio.h>
int main()
{
	printf("Hello world!"\n);
}
```
- По умолчанию `printf()` выводит строку в консоль, но порой нам надо вывести в консоль какую-то переменную, тип данных которой отличен от строки. В таком случае синтаксис функции `printf()` приобретает следующий вид:
```
printf("спецификатор", переменная);
```
- В примере выше первым аргументом та же строка в кавычках, только теперь мы указываем экранирование строки с помощью спецификатора. Это значит, что мы с помощью определенных служебных символов управляем выводом переменных с типом данных, отличных от строки. Про спецификаторы можно прочитать ниже. Причем в строке можно указывать сколь угодное количество спецификаторов, чтобы вывести в консоль такое же количество переменных:
```
#include <stdio.h>

int main()
{
	int day = 02;
	int month = 11;
	int year = 2007;
	
	printf("%d.%d.%d", day, month, year); // output: 02.11.2007
	
	return 0;
}
```
- Для ввода уже нужно объявить переменную, куда будет сохранятся информация, а также понять, какое количество информации мы хотим получить. Пока просто сохраним какое-нибудь число:
```
#include <stdio.h>
int main()
{
	int x;
	scanf("%<спецификатор>", &x);
}
```
- Суть такая же, что и с выводом. Мы указываем спецификатор, чтобы компилятор правильно понял, какого типа данных мы хотим принять информацию, а затем пишем указатель на переменную, куда эту самую информацию сохраним. Причем как с выводом, так и с вводом, мы можем указать сколь угодное количество принимаемой информации (в рамках разумного).
****
### **Спецификаторы формата ввода**
- `%d или %i:` Целое число со знаком
- `%u`: целое число без знака
- `%f`: число с плавающей запятой (формат по умолчанию)
- `%e или %E`: числа с плавающей запятой в экспоненциальном представлении
- `%g или %G`: числа с плавающей запятой в фиксированной или экспоненциальной записи (более короткий формат)
- `%c`: отдельный символ
- `%s`: Строка
- `%p`: Указатель на адрес
- `%%`: Буквальный % символ

### **Ветвления и циклы**
- Для начала надо понять, что логические условия принимают только два значения - истина или ложь, 1 или 0, `true` или `false`. 
- Рассмотрим ветвления. Если на основе какого-то условия требуется выполнить определенные действия, то пишем логическую конструкцию `if`. Выполняется, если условие истинно, если же нет, то дописываем `else`.  Кстати, подобные проверки можно плодить сколько угодно. Вот пример всех трех случаев:
```
if (condition) { // тут какой-то код; }

if (condition)
{
	// тут тоже код;
}
else
{
	// а тут код, если иное не сработало;
}

if (condition1) { // ну и так далее; }
else if (condition2) { ... }
else if (condition3) { ... }
else { ... }
```
- Стоит заметить, что можно комбинировать условия с помощью логических операторов: `&&` - логическое "И", `||` - логическое "ИЛИ", `!` - логическое "НЕ", `==` - логическое сравнение, ну или конъюнкция, дизъюнкция, инверсия и эквиваленция. Ну и пример:
```
bool a = true;
bool b = false;

if (a && b) { return 1; } else { return 0; } // 0
if (a || b) { return 1; } else { return 0; } // 1
if (!a) { return 1; } else { return 0; } // 0
if (!b) { return 1; } else { return 0; } // 1
if (a == b) { return 1; } else { return 0; } // 0
```
- Теперь ветвление `switch case`. Тут проще, так как мы задаем переменную, а на основе ее значения уже выполняем операции. 
```
int x;
switch (x)
{
	case 1:
		// какой-то код
		break;
	case 2:
		// какой-то код
		break;
	case 3:
		// какой-то код
		break;
}
```
- Кстати, `break;` надо обязательно писать в конце каждого блока `case`. 
- Цикл `while-do` сначала проверяет выполнение какого-то условия и пока оно работает, цикл будет работать. 

- Цикл `do-while` сначала выполняет какое-то действие, а потом проверяет некое условие, и пока оно истинно, будет выполнять его и дальше. 

- Цикл `for` это цикл со счетчиком. Мы задаем переменную, дальше сравниваем ее с неким условием, а потом изменяем эту переменную. Пока условие истинно, цикл работает.

### **Прототипы функций**
- Компилятор С проходится по всей программе и проверяет весь код на синтаксические ошибки. В разработке выходит так, что сначала мы объявляем функции, а потом используем их в функции `main()`, чтобы ошибок не возникало, иначе сначала мы используем созданную нами функцию, а потом уже объявляем ее. В таком случае компилятор выдаст ошибку. 
- Но можно этого избежать, если создать прототип функции. Суть такая. Перед тем, как полноценно написать функционал функции, мы можем создать ее прототип, и компилятор "будет уведомлен" о наличии нужной функции. Делаем так:
```
#include <stdio.h>

// описание
void hello();

int main(){
	hello();
	return 0;
}

void hello(){
	printf("Hello\n");
}
```
- Сначала мы дали компилятору понять, что функция `hello()` существует, а уже после функции `main()` описали ее поведение.
